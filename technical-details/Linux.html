Linux Guides

 

General

It is generally advised to use Proton GE for ChilloutVR.

 

Issues&Notices

ChilloutVR is a Platform driven by user generated Content, with that comes the typical issues like on the fly Shader Recompilation.
 
ChilloutVR is a Mono game that makes use of Burst/Jobs/Collections aka DOTS (mostly for 3rd party implementations), this drastically improves performance but Native Machine Code is simply more painfull to translate and Proton/Wine can fail, this is usally indicated by Memory Leaks and Crashes Related to Memory Exeptions and Out of Bounds.

 

VR

It is advised to refrain from using SteamVR or OpenVR directly, the advised setup is Monado/WiVRn with OpenComposit.

WiVRn setup:

  • You should first try the Offical WiVRn Setup.


If for any reason this is not applicable with your setup, follow the next steps.

 

WiVRn through Envision setup:

  • Get Envision through the package Manager of your choice.
     
  • now you see Envision profiles in a dropdown at the bottom, you can select WiVRn from there.
     
  • hit the Build Profile Button.
     
  • if everything goes smooth, you should now be able to Launch wivrn from the Envision UI, enter the paircode in your headset and connected.


    If everything dosnt go smooth:
  • copy the WiVRn profile in Envision.
     
  • edit the duplicated profile.
     
  • You need compile this fork: https://github.com/notpeelz/WiVRn.git, branch solarxr-patches. You can easily build WiVRn from this specific branch via Envision, by modifying “XR Service Repo” and “XR Service Branch” in your build profile accordingly.
     
  • hit the Build Profile again. (if you encounter a build issue with “Eigen3” use your package manager of choice to install it then rerun the build process)
